# 画面説明: GODUFO Portfolio

実装設計の詳細は DESIGN.md に分離し、この文書では画面構成、操作導線、見せ方に限定して説明する。

## 1. 目的
- 開発目的: ブラウザ 2D シューティングを題材に、プレイ体験を壊さずに責務分離できることを示す。
- 評価してほしい点: 1 画面構成の中で、開始導線、プレイ導線、結果確認が分かりやすく整理されていること。

## 2. 画面構成

### 2.1 タイトル表示
- 対象: index.html 読み込み直後のオーバーレイ
- 主な要素:
  - タイトルロゴ
  - INITIALIZE MISSION ボタン
  - 操作導線の導入表示
- 役割: 初見ユーザーに開始ポイントを一つだけ提示し、迷わずプレイへ入れるようにする。

### 2.2 プレイ画面
- 対象: タイトル解除後のメイン表示
- 主な要素:
  - Canvas に描画される自機、敵、弾、演出
  - SCORE 表示
  - WAVE 表示
  - SHIELD ゲージ
  - SPECIAL ボタン
- 役割: 視線を中央のプレイ領域に集めつつ、右上や上部で進行状況を即時確認できるようにする。

### 2.3 終了状態
- 対象: シールド消失後またはゲーム終了時
- 主な要素:
  - 結果表示
  - リトライ導線
- 役割: プレイ終了を明確に伝え、再挑戦の流れを途切れさせない。

## 3. 操作導線
- 開始: タイトル画面のボタン押下でゲーム開始。
- 移動: マウスまたはタッチ位置へ自機を移動。
- 攻撃: 押下中に溜め、離したタイミングで発射。
- 必殺技: 画面ボタン、Space、X で発動。
- 再挑戦: 終了後に再度プレイへ戻れる構成。

## 4. 画面上の見どころ
- HUD は SCORE、WAVE、SHIELD、SPECIAL の 4 要素に絞り、短時間評価でも追いやすくしている。
- 開始前とプレイ中の状態差を大きくし、どこが操作可能かを見た瞬間に分かるようにしている。
- スマホではタッチ操作と画面ボタンを優先し、デスクトップではマウスとキーボードも併用できる。
